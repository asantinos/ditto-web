## Before you start

Two things should be in place from the previous steps:

- Ditto is installed and running. The pill is somewhere on screen, or hidden in the tray.
- A Whisper model is downloaded. If you skipped the welcome window, open **Settings → Models** and pick one.

<Callout type="tip">
This walkthrough assumes the default hybrid shortcut <kbd>Ctrl</kbd>+<kbd>Windows</kbd>. If you changed it, use yours.
</Callout>

## Dictating

1. **Click into a text field.** Anywhere a cursor blinks works: a Slack message, a code editor, an email draft, a search bar. Ditto pastes wherever your cursor is when transcription finishes.
2. **Hold <kbd>Ctrl</kbd>+<kbd>Windows</kbd>.** The pill switches to its recording state. On Matte and Onyx themes it shows voice-driven bars; on Aurora, Sunset, and Mint it shows a wave at the bottom. Silero VAD keeps the visual response focused on detected speech.
3. **Talk normally.** Speak at your usual pace. Pauses are fine, Whisper will handle them. Background noise is OK in moderation, especially with the noise filter on.
4. **Release the keys.** The pill collapses to a small dots loader while Ditto runs the audio through Whisper and, if enabled, the local Qwen formatter.
5. **Your text is pasted.** Within a second or two (longer for big models, long recordings, or smart formatting), the final text appears in the field where your cursor was and is added to local history if history is enabled.

To dictate hands-free, quickly press <kbd>Ctrl</kbd>+<kbd>Windows</kbd> twice. Press it once more to finish and transcribe.

## What each pill state looks like

| State | What you see | Meaning |
|---|---|---|
| Idle | Thin bar that expands to "Ditto" on hover | Ready, waiting for the shortcut |
| Recording | Voice-reactive bars or wave | Capturing audio |
| Transcribing | Small pill with a row of animated dots | Running Whisper, generating text |
| Copied | Short pill with a check icon and "Copied!" | Text is in your clipboard but auto-paste is off |
| Error | Flat micro-pill with a red dot and message | Microphone access or transcription failed |

The "Copied" state only appears if you turned off auto-paste in **Settings → General**. By default Ditto pastes the text for you.

## Canceling a recording

If you change your mind mid-sentence:

- Press <kbd>Esc</kbd> while recording. The pill returns to idle and the audio is discarded. Nothing is sent to Whisper.

The cancel shortcut is configurable in **Settings → Shortcut**, in case <kbd>Esc</kbd> clashes with something else for you.

## If something goes wrong

A few common situations:

- **Nothing happens when you press the shortcut.** Another app may have registered the same combo. Open **Settings → Shortcut** and pick a different one, or check the [Shortcuts page](/docs/shortcuts) for details.
- **The pill records but no text appears.** Check that the model file still exists. If you deleted it manually, Ditto reopens the welcome window so you can pick another. See [Models](/docs/models) for more.
- **The text is wrong or in the wrong language.** Open **Settings → Audio** and set a specific transcription language instead of "Auto-detect". See [Audio input](/docs/audio-input) and [Languages](/docs/languages).
- **The pill never reaches "Copied" and your app is empty.** Some apps (RDP sessions, certain games, sandboxed text fields) block simulated <kbd>Ctrl</kbd>+<kbd>V</kbd>. Turn on **Settings → General → Keep previous clipboard** if you'd rather paste manually.

For broader issues, jump to [Troubleshooting](/docs/troubleshooting).

## A few tips for better results

- **Let smart formatting handle structure.** When enabled, the local Qwen model can add punctuation, paragraphs and lists and remove filler words without sending text anywhere.
- **Don't whisper.** A normal speaking voice gives the model the most signal.
- **Bigger isn't always better.** Base or Small are usually plenty for daily notes. Medium and Large-v3 shine for technical jargon, accents, or noisy environments — but they're slower.
- **Pick a fixed language if you always speak the same one.** Auto-detect is flexible but adds latency. Setting it explicitly speeds things up and reduces mistakes.
